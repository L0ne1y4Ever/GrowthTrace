## 2. 接口清单

建议保留以下接口作为课程报告和答辩依据：

- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/auth/me`
- `POST /api/profile/onboarding/extract`
- `POST /api/profile/onboarding/confirm`
- `GET /api/profile`
- `GET /api/target`
- `POST /api/target/{id}/requirements`
- `GET /api/journal`
- `POST /api/journal/{id}/extract`
- `POST /api/journal/{id}/extract/confirm`
- `GET /api/execution/task`
- `POST /api/execution/task/{id}/check-in`
- `POST /api/diagnosis/trigger`
- `GET /api/diagnosis/history`
- `GET /api/dashboard/overview`

## 3. 表结构摘要

建议在报告中重点展示以下核心表：

- `sys_user`
- `growth_profile`
- `profile_skill`
- `profile_experience`
- `growth_target`
- `target_requirement`
- `growth_journal`
- `journal_extraction`
- `growth_task`
- `stage_assessment`
- `growth_snapshot`

## 4. 演示路径

推荐采用以下演示顺序：

1. 用户注册/登录
2. 成长建档
3. 查看画像
4. 设定目标与 requirement
5. 创建执行任务
6. 编写随记并触发 AI 抽取
7. 确认抽取结果
8. 触发阶段诊断
9. 查看 Dashboard
