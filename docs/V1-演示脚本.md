# GrowthTrace V1 演示脚本

## 1. 演示目标

用 5-8 分钟展示 GrowthTrace 的完整业务闭环，证明系统具备：

- 学生成长数据沉淀能力
- AI 辅助抽取与诊断能力
- 目标到执行的闭环能力
- 可视化展示能力

## 2. 演示环境

- 后端：`http://localhost:8080/api`
- 前端：`http://127.0.0.1:5173`
- 数据库：本机 MySQL `growthtrace`
- 演示账号：`demo / password`

## 3. 推荐演示顺序

### 3.1 登录

- 使用 demo 账号登录
- 说明系统采用 JWT 鉴权
- 登录后进入 Dashboard

### 3.2 Dashboard 总览

重点讲 4 个点：

- 画像完整度
- 当前主目标
- 近期随记 / 诊断摘要
- 热力图与成长曲线

讲解口径：
Dashboard 不是静态看板，而是对建档、目标、执行、诊断结果的统一聚合。

### 3.3 成长档案

- 打开 Profile 页面
- 展示个人画像摘要、技能、经历
- 说明档案不是 AI 直接入库，而是“抽取草稿后由用户确认”

### 3.4 目标设定

- 打开 Target 页面
- 展示主目标与目标要求
- 说明要求可以拆成技能、知识、经历等维度

### 3.5 成长随记

- 打开 Journal 页面
- 展示已有随记
- 进入详情页，说明系统支持对自然语言随记做结构化抽取

如现场有 AI 密钥：

- 演示一次抽取草稿
- 展示用户确认后再入档

如现场无 AI 密钥：

- 直接说明抽取链路已预留，当前演示使用 seed 数据展示结果页

### 3.6 执行任务

- 打开 Execution 页面
- 展示任务看板、状态流转、打卡与周进度
- 强调系统不是只给建议，也追踪执行落地

### 3.7 阶段诊断

- 打开 Diagnosis 页面或历史页
- 展示阶段总结、关键问题、建议、纠偏方向、复盘内容
- 说明诊断基于已有成长数据与阶段窗口生成

## 4. 收尾话术

建议用下面这个结构收尾：

1. GrowthTrace 完成了从建档到目标、从记录到诊断、从建议到执行的闭环
2. V1 优先保证可实现、可部署、可演示，而不是追求复杂模型堆叠
3. 后续 V2 才考虑多模型切换、缓存优化、成长计划分组等增强能力

## 5. 现场风险与兜底

### 5.1 AI 接口不可用

- 直接切 demo 数据
- 重点展示已抽取后的确认结果、诊断历史和 Dashboard

### 5.2 前端未启动

- 先用后端接口或 Swagger 类工具做接口说明
- 再补一句前端构建已通过

### 5.3 网络异常

- 本项目核心演示基于本机数据库和本地服务
- 除 AI 依赖外，其余链路都可离线展示
